Recent Happenings:

Just today (June 30th, 2017), we finished our side-scrolling game! Developed by Hannah, Berenice, and myself, it's called "Under the Sea" and features a diver as the character. The person playing needs to avoid obstacles like fish, sharks, and seaweed, while aiming to get air bubbles to increase his or her air supply. When the time (increased by getting air bubbles) hits zero, you lose! The object of the game is to continually get a higher score every time you play.

Projects

Obamicon: Back in 2008, Shepard Fairey designed a poster of Barack Obama that used only red, blue, and beige. It's now a global icon; we took Fairey's idea and created a code that would take a picture and convert it to red, blue, and beige based on the intensity of the pixel's color. The Obamicon project focused on loops and conditionals in Python; one hugely helpful part of the whole process was writing up a pseudocode before going technical.

Drawing Shapes in Python and Scratch: This code was designed to take in an input (the number of sides for the computer to draw) and draw a shape based on the input, which would be the output. This was easier to do in Scratch than in Python; in Scratch, all the code available was there in the program, and all I had to do was drag and drop the correct block of code into the workspace. It was logical and pretty straightforward. In Python, I had to type up the entire code, and all I had were a few lines of code that I had to string together.

Paddleball: The paddleball game consists of two people playing on the computer using the right and left arrow keys and the A and D arrow keys; the goal is to keep the ball bouncing on the platform. It took a long time to get the ball to bounce off the platform and the walls; the easier part was designing the score variable. If I had had more time, I would've wanted to focus on the physics of the ball, so that it would look more realistic.

Robots: Switching from Scratch and Python to robots and Arduino was a big change; suddenly, we had to learn all of the hardware in a robot in addition to a new programming language. Wiring the robot took some time getting used to, and sometimes the robot wouldn't work because of the wiring, and not because of the code. Sometimes, a piece would blow out (like an LED) and we wouldn't be able to use it, but we would spend a long time laboring over the code and the wiring before deciding to try switching out the LED. More frustration equals more excitement when it works, though, and in the end, it was like the robot was a lil' baby.
